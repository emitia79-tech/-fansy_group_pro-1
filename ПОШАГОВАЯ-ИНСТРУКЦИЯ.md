# Пошаговая инструкция: бот Fansy на fancy-group.pro

Селектор кнопки «Чат» уже известен: **`.crm-widget-button-mobile`**. Остаётся развернуть бота и вставить один блок кода на сайт.

---

## ЧТО КУДА ПОДСТАВИТЬ (кратко)

| Где | Что подставить |
|-----|----------------|
| **Render.com** → Environment | `OPENAI_API_KEY` = ваш ключ OpenAI (sk-proj-...) |
| **Файл «КОД-ДЛЯ-САЙТА.txt»** | Заменить `https://ВАШ_СЕРВЕР_БОТА` на URL из Render (например https://fansy-chat-xxxx.onrender.com) |
| **Админка fancy-group.pro** | Вставить содержимое «КОД-ДЛЯ-САЙТА.txt» (после подстановки URL) в раздел «Дополнительный код» / «Код перед </body>» |

---

## ЧТО ГДЕ СОЗДАТЬ

| № | Где | Что создать/сделать |
|---|-----|---------------------|
| 1 | **GitHub** | Аккаунт (если нет) → репозиторий → загрузить папку проекта (без api_key.txt) |
| 2 | **Render.com** | Аккаунт → New → Web Service → подключить репо → настроить Build/Start и переменную OPENAI_API_KEY |
| 3 | **Файл на ПК** | Открыть «КОД-ДЛЯ-САЙТА.txt», заменить URL, скопировать весь блок |
| 4 | **fancy-group.pro** | В админке вставить скопированный код в раздел с дополнительным кодом страницы |

---

## ШАГ 1. Репозиторий на GitHub

1. Зайдите на https://github.com и войдите (или зарегистрируйтесь).
2. Нажмите **New** (или **Create repository**).
3. Имя репозитория: например `fansy-bot`. Public. **Create repository**.
4. На ПК откройте папку проекта `Проект Fansy.ru`.
5. Инициализируйте git и залейте код (в PowerShell или терминале):

```bash
cd "c:\Desktop\Проекты Cursor\Проект Fansy.ru"
git init
git add .
git commit -m "Fansy bot initial"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/fansy-bot.git
git push -u origin main
```

(Замените `ВАШ_ЛОГИН` и `fansy-bot` на ваш логин и имя репо.)

**Важно:** файл `api_key.txt` в проекте в `.gitignore` — в репозиторий не попадёт. Ключ будете задавать на Render.

---

## ШАГ 2. Сервис на Render.com

1. Зайдите на https://render.com и войдите (или зарегистрируйтесь через GitHub).
2. **New** → **Web Service**.
3. Подключите ваш GitHub-репозиторий с проектом Fansy (если ещё не подключён — **Connect account** и выберите репо).
4. Укажите:
   - **Name:** `fansy-chat` (или любое).
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn server:app --host 0.0.0.0 --port $PORT`
5. В блоке **Environment** нажмите **Add Environment Variable**:
   - **Key:** `OPENAI_API_KEY`
   - **Value:** ваш ключ OpenAI (начинается с `sk-proj-...`). Поставьте галочку **Secret**.
6. Нажмите **Create Web Service**. Дождитесь окончания сборки (несколько минут).
7. Вверху страницы появится **URL** вида `https://fansy-chat-xxxx.onrender.com`. **Скопируйте его** — это и есть «ВАШ_СЕРВЕР_БОТА».

---

## ШАГ 3. Подставить URL в код для сайта

1. На ПК откройте файл **`КОД-ДЛЯ-САЙТА.txt`** (он лежит в папке проекта).
2. Замените в нём **все** вхождения `https://ВАШ_СЕРВЕР_БОТА` на ваш URL с Render (без слэша в конце).  
   Пример: было `https://ВАШ_СЕРВЕР_БОТА` → стало `https://fansy-chat-xxxx.onrender.com`.
3. Скопируйте **весь** содержимое файла (оба тега `<script>...</script>`).

---

## ШАГ 4. Вставить код на fancy-group.pro

1. Войдите в админку сайта https://fancy-group.pro (панель управления).
2. Найдите раздел для дополнительного кода страницы. Обычно это:
   - **Настройки сайта** → **Дополнительный код** / **Код перед </body>** / **Скрипты в подвале**, или
   - **Внешние скрипты** / **Виджеты** / **HTML-вставка**.
3. Вставьте туда скопированный из **КОД-ДЛЯ-САЙТА.txt** блок (целиком).
4. Сохраните и опубликуйте изменения.

---

## ШАГ 5. Проверка

1. Откройте в браузере https://fancy-group.pro (лучше в режиме инкогнито или с очисткой кэша).
2. Нажмите на кнопку «Чат» (та, что с классом `crm-widget-button-mobile`).
3. Должна открыться панель с заголовком «Fansy» и полем ввода. Напишите сообщение и отправьте — ответ должен прийти от бота.

Если панель не открывается или запрос не уходит — откройте F12 → вкладка **Console** и **Network** и посмотрите, нет ли ошибок или блокировки запроса (CORS). Убедитесь, что в коде подставлен именно URL вашего сервиса с Render.
